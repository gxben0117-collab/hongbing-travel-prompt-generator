# 文件流程

## 每次更新前

1. 先確認目前最新版本，例如 `V1.00`。
2. 新版本加 `0.01`，例如下一版為 `V1.01`。
3. 不覆蓋舊版紀錄檔。

## 每次更新後

新增一份開發紀錄：

```text
docs/development-log/V1.01.md
```

紀錄內容建議包含：

- 更新日期
- 更新目標
- 修改檔案
- 完成內容
- 後續待辦

## 專案文件維護

- `docs/project/專案總覽.md` 記錄專案整體狀態。
- `docs/版本規則.md` 記錄版本命名與累積規則。
- `docs/development-log/` 保存每一次更新紀錄。
