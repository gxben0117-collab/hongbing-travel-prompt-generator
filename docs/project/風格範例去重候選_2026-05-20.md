# 風格範例去重候選

日期：2026-05-20

## 本輪目的

針對 `紅兵旅拍大片生成器` 的旅拍模式風格範例，先整理出「重複性高、容易互相吃掉存在感」的候選群組，作為下一輪精簡依據。

目前先做候選整理，不直接刪，避免一次砍掉過多有效變化。

## 初步重複熱區

依 `src/data.js` 目前 `img_ref` 與相近大型 preset 觀察，重複風險較高的群組集中在：

- `wedding`
- `editorial`
- `tang`
- `imperial_empress`
- `fallen_angel`

## 高重複候選群組

### 1. 墮天使 / 暗黑聖堂系

候選：

- `ref_fallen_angel_ruins`
- `ag_fallen_angel_darkest`
- `ag_fallen_angel_darkest`
- `fa_lucifer_cathedral`
- `fa_crimson_halo`
- `fa_dark_coronation`
- `fa_moonlit_ruin_peace`
- `fa_sorrow_throne`

原因：

- 都圍繞黑白羽翼、聖堂廢墟、黑金 / crimson 光影、悲劇神聖感
- 若以一般使用者角度看，很容易感覺只是同題材不同站姿

### 2. 唐代宮廷 / 皇后 / 貴妃系

候選：

- `ref_tang_empress_peony`
- `ref_tang_consort_lantern`
- `hf_tang_empress_gold`
- `hf_tang_peony_consor`
- `hf_tang_hu_dance`
- `hf_tang_night_lantern`
- `hanfu`
- `guifei`

原因：

- 都是 Tang court / peony / lantern / imperial silk / palace glow
- 目前數量偏多，容易出現同質華麗感堆疊

### 3. 清宮 / 格格 / 太后 / 皇后系

候選：

- `ref_qing_gege_snow_garden`
- `ref_qing_empress_throne`
- `hf_qing_empress_formal`
- `hf_qing_gege_winter`
- `hf_qing_flower_pavilion`
- `qing_palace`

原因：

- 都是清宮旗裝、宮牆、雪庭 / 皇殿 / 珠冠 / 旗頭語彙
- 可保留雪景格格、朝座太后、花房柔系三支代表，但不宜再保留過多近似衍生

### 4. 婚紗 / 婚禮 / 珠寶盒 / 長階月夜系

候選：

- `ref_castle_wedding_princess`
- `ref_jewel_music_box_champagne`
- `ref_midnight_wedding_stairs`
- `wedding`
- `ref_white_scarf_lake_goddess`

原因：

- 都有白紗、浪漫月夜 / 庭園 / 城堡 / 高訂婚紗語彙
- 其中部分其實只是背景或道具略變

### 5. 世界景點 / 藍城 / 城市旅拍 editorial 系

候選：

- `ref_world_landmark_morocco`
- `wc_morocco_blue_riad`
- `ref_city_gold_dress_cat`
- `ref_beauty_life_cover`
- `ref_afternoon_tea_salon`

原因：

- 都偏 lifestyle / editorial / destination travel
- 有些更像場景換皮，而不是明確不同風格範例

## 建議下一步

1. 每個高重複群組先保留 2 到 3 支代表款
2. 優先刪除：
   - 名稱語感接近
   - 光影語彙接近
   - 姿勢與服裝語彙只小改
3. 真正刪除前先比對：
   - `src/data.js`
   - `index.template.html`
   - 是否有 UI 卡片直接引用

## 目前建議策略

不要一次整片砍。

最穩定做法是：

- 每輪只清一個高重複群組
- 先從 `img_ref` 裡最明顯的同題材變體下手
- 每次刪完就 build / 驗證

## 本輪已執行

已先移除兩個高重複、替代性高的 `img_ref`：

- `ref_beauty_life_cover`
- `ref_afternoon_tea_salon`
- `ref_jewel_music_box_champagne`
- `ref_castle_wedding_princess`

原因：

- 兩者都偏 lifestyle / 雜誌 / 甜點沙龍感
- 與現有 `soft_daily`、`editorial`、`wedding` 鄰近參考在使用者感知上重疊度高
- 刪除後仍保留足夠的高級旅拍、生活感與婚紗系參考，不會造成明顯斷層

補充：

- `ref_jewel_music_box_champagne` 雖然華麗，但更偏禮盒童話 / 高級甜夢商品櫥窗感
- 與目前保留的婚紗 / 童話 / 奢華甜美群有高重疊
- 對旅拍主題的辨識度貢獻低於其佔用的範例空間

- `ref_castle_wedding_princess` 雖然完成度高，但偏童話婚禮公主主視覺
- 與 `ref_midnight_wedding_stairs` 在使用者感知上同屬高級婚紗主視覺
- 相比之下，`午夜婚紗長階` 對旅拍大片的場景辨識度更高，因此優先保留後者
